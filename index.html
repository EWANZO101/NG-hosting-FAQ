Getting Started

Your hosting provider will give you access details including:

  • Panel URL: Usually something like https://panel.yourhost.com
  • Username/Email: Your login credentials
  • Password: Your account password

Important: Always change the default password after your first login for security.

Step 1: Navigate to Server Creation

From your dashboard, look for a "Create Server" or "+" button. This is usually prominently displayed.

Step 2: Choose Server Type

Select the appropriate egg/template for your game:

  • Minecraft (Java/Bedrock)
  • FiveM
  • Counter-Strike
  • Generic (for custom applications)

Step 3: Configure Basic Settings

  • Server Name: Choose a descriptive name
  • Memory: Allocate RAM based on your plan
  • Disk Space: Set storage limits
  • CPU: Set processing power limits

The file manager is your primary tool for uploading, editing, and organizing server files.

Basic Operations

  • Upload files via drag & drop
  • Edit text files directly in browser
  • Create new folders and files
  • Download files to your computer
  • Set file permissions

Advanced Features

  • Archive extraction (.zip, .tar.gz)
  • Mass file operations
  • File search functionality
  • Backup and restore
  • Version control for configs

2

Edit eula.txt file and change "false" to "true"

3

Configure server.properties

Set server name, difficulty, game mode, etc.

4

Start Server

Click the start button and monitor console output

Memory Allocation Guidelines

  • 1-5 players: 2GB RAM minimum
  • 5-15 players: 4GB RAM recommended
  • 15-30 players: 6-8GB RAM
  • 30+ players: 8GB+ RAM with optimization

Installing Plugins (Spigot/Paper)

  1. Download plugin .jar file from trusted sources
  2. Upload to the /plugins/ folder
  3. Restart server or use /reload command
  4. Configure plugin settings in /plugins/[plugin-name]/

Popular Sources: SpigotMC, Bukkit, GitHub

Installing Mods (Forge/Fabric)

  1. Ensure server is running Forge/Fabric
  2. Download compatible mod .jar files
  3. Upload to the /mods/ folder
  4. Restart server to load mods

Popular Sources: CurseForge, Modrinth

Server Configuration

1. Server.cfg Setup

Edit your server.cfg file with these essential settings:

# Basic Settings
endpoint_add_tcp "0.0.0.0:30120"
endpoint_add_udp "0.0.0.0:30120"
sv_hostname "Your Server Name"
sv_maxclients 32
sv_licenseKey "your_license_key_here"
2. Resource Management

Add resources to your server.cfg:

# Essential Resources
start mapmanager
start chat
start spawnmanager
start sessionmanager
start basic-gamemode

Resource Installation

  1. Download resources from trusted sources (FiveM forums, GitHub)
  2. Extract to /resources/ folder
  3. Add "start [resource-name]" to server.cfg
  4. Restart server to load new resources

Troubleshooting

Common Startup Issues

1. Check Console Logs

Look for error messages in the console tab - they usually indicate the exact problem.

2. Verify EULA Acceptance

For Minecraft servers, ensure eula.txt contains "eula=true".

3. Check Resource Allocation

Ensure sufficient RAM and disk space are allocated.

4. Configuration File Syntax

Check for typos or formatting errors in config files.

Quick Diagnostic Steps

  1. Check if server files exist and aren't corrupted
  2. Verify Java version compatibility (for Minecraft)
  3. Test with minimal configuration
  4. Check port availability and firewall settings
  5. Review recent file changes

Minecraft Optimization

  • Use Paper/Purpur: Better performance than Spigot
  • Optimize server.properties: Reduce view distance, entity limits
  • Install optimization plugins: LimitPillagers, EntityCleaner
  • Regular world maintenance: Use WorldBorder, prune chunks
  • JVM flags: Use Aikar's flags for better garbage collection

FiveM Optimization

  • Resource management: Remove unused resources
  • Optimize scripts: Use ox_lib for better performance
  • Database optimization: Index frequently queried columns
  • Limit OneSync: Use appropriate OneSync level
  • Monitor resource usage: Use resmon to identify heavy resources
